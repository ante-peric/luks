3.2. Brisanje LUKS Particije


Nakon što smo kreirali LUKS particiju I uspješno je mountali sad je možemo I brisati na način:


Provjeravamo status LUKS diska/particije


# cryptsetup status secret

/dev/mapper/secret is active and is in use.

type: LUKS1

cipher: aes-xts-plain64

keysize: 512 bits

key location: dm-crypt

device: /dev/mapper/vg_luks-lv_luks

sector size: 512

offset: 4096 sectors

size: 2084864 sectors

mode: read/write



umount


umount direktorij na način da umountamo mount točku imena /secret


# umount /mnt/secret


Ili


umount LUKS enkriptirani file system prema LUKS imenu


umount /dev/mapper/secret


  • Brišemo /etc/fstab zapis – vezan za LUKS

  • Brišemo brišemo sadržaj file – a /etc/crypttab.

  • Brišemo postojeće ključeve iz particije ili diska:


# cryptsetup luksRemoveKey /dev/vg_luks/lv_luks


Enter passphrase to be deleted: Unesite zaporku ovdje







Zaustavljamo LUKS uređaj


# cryptsetup luksClose secret


Brišemo LUKS uređaj


[root@peric ~]# cryptsetup remove /dev/mapper/secret


Device /dev/mapper/secret is not active



Sljedeći korak je brisanje: LV


# lvremove /dev/vg_luks/lv_luks


Do you really want to remove active logical volume vg_luks/lv_luks? [y/n]: y

Logical volume "lv_luks" successfully removed


Zatim brišemo: VG


# vgremove /dev/vg_luks


Volume group "vg_luks" successfully removed


Zatim brišemo: PV


# pvremove /dev/sdc1


Labels on physical volume "/dev/sdb1" successfully wiped.


Ako želimo USB restartirati / brisati sve LVM zapise :


wipe LVM iz particije/diska


# wipefs -a /dev/sdc1

# wipefs -a /dev/sdc

/dev/sdb: 2 bytes were erased at offset 0x000001fe (dos): 55 aa

/dev/sdb: calling ioctl to re-read partition table: Success


# partprobe


Zatim pomoću disk-utility fdisk brišemo particiju (ovo nije prikazano – mora se znati).


Ako više ne želimo za u buduće koristiti etc/luks-leys/… možemo izbrisat file – e ili cijeli direktorij


# rm -rf /etc/luks-keys
